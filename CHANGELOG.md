# سجلّ التغييرات

تلتزم هذه الإضافة بـ[الإصدار الدلاليّ](https://semver.org/lang/ar/).

## [0.2.0] - 2026-06-20

### أُضيف
- **استراتيجية `webview` الآمنة (الافتراضيّة الآن):** سطح RTL أصليّ داخل لوحة
  webview مدعومة رسميًّا، دون تعديل ملفّات تثبيت VS Code، يصمد عبر التحديثات.
- إعداد `vsCodeRtl.hideNativeChrome`: إخفاء شريط الأنشطة/الحالة الأصليّين عبر
  إعدادات رسميّة قابلة للعكس (`LayoutManager`).
- توليد HTML نقيّ قابل للاختبار لسطح RTL (`webviewHtml.ts`) مع CSP وتهريب آمن.
- نوع انحراف `reopen`: إعادة فتح أسطح webview تلقائيًّا عند الإقلاع (لأنّها لا تصمد عبر إعادة التحميل).

### تغيّر
- الاستراتيجية الافتراضيّة من `cssInjection` إلى `webview` الآمنة؛ صارت
  `cssInjection` خيارًا متقدّمًا.
- كشف الانحراف صار واعيًا بالاستراتيجية (`survivesReload`).
- ارتفع عدد اختبارات الوحدة إلى 32.

## [0.1.0] - 2026-06-20

### أُضيف
- بنية استراتيجيات قابلة للتوسّع (`RtlStrategy` + `StrategyRegistry`).
- استراتيجية `cssInjection` لتطبيق RTL عبر حقن CSS في واجهة VS Code.
- الأوامر: تفعيل، تعطيل، تبديل، إعادة التطبيق.
- إعدادات: الاستراتيجية، النطاق (`workbench`/`all`)، إبقاء المحرّر LTR.
- تتبّع الحالة في `globalState` عبر `MementoState`.
- كشف الانحراف عند الإقلاع: اقتراح إعادة التطبيق إن محا تحديثُ VS Code الحقنَ.
- مؤشّر شريط الحالة يعرض الوضع ويبدّله بالنقر.
- نسخة احتياطية للحالة الأصليّة قبل أوّل حقن.
- اختبارات وحدة (26) واختبارات تكامل داخل VS Code.
